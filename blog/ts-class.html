TypeScript类

April 28, 2021 · 4 min read

ts类的学习记录

类定义​

class Person {
    name: string
    constructor(name: string) {
        this.name = name
    }
    sayName() {
        console.log(this.name)
    }
}
const p = new Person('Steven')
console.log(p.name) // Steven
p.sayName()  // Steven

继承​

class Person {
  name: string
  constructor(name: string) {
    this.name = name
  }
  sayName() {
    console.log(this.name)
  }
}
class Student extends Person {
  name: string
  constructor(name) {
    super(name)
  }
  sing() {
    console.log(`${this.name} singing now!`)
  }
}
const p = new Student('Steven')
console.log(p.name) // Steven
p.sing()  // Steven singing now!

访问修饰符​

在 ts 中可以使用三种访问修饰符，分别是 public、private 和 protected

  • public: 修饰的属性或方法是公有的，可以在任何地方被访问到，默认所有的属性和方法都是 public
  • private：修饰的属性或方法是私有的，不能在声明它的类的外部访问
  • protected：修饰的属性或方法是受保护的，它和 private 类似，但是它在子类中是允许被访问的
class Person {
    public name: string
    public constructor(name: string) {
        this.name = name
    }
    public sayName() {
        console.log(this.name)
    }
}
const p = new Person('Steven')
p.sayName()

private 私有属性，只能在类中访问的属性，子类与实例中均无法访问

class Person {
    private name: string
    public constructor(name: string) {
        this.name = name
    }
    public sayName() {
        console.log(this.name)
    }
}
const p = new Person('Steven')
console.log(p.name)
p.sayName()

class Student extends Person {
    constructor(name) {
        super(name)
        console.log(this.name)  // 属性“name”为私有属性，只能在类“Person”中访问。ts(2341)
    }
}

protected 跟 private 相似，只能在类中使用，但是 protected 可以在继承的子类中被访问

class Person {
  protected name: string
  public constructor(name: string) {
    this.name = name
  }
  public sayName() {
    console.log(this.name)
  }
}

class Student extends Person {
  constructor(name) {
    super(name)
    console.log(this.name) // 子类中可以正常访问
  }
}

const p = new Student('Steven')
console.log(p.name) // 属性“name”受保护，只能在类“Person”及其子类中访问。ts(2445)
p.sayName()

静态属性​

只能通过类直接访问的属性或方法，实例无法调用。

class Person {
  private name: string
    static age: number = 20 // 静态属性
  constructor(name) {
    this.name = name
  }
  sayName() {
    console.log(this.name)
  }
  // 通过 static 定义静态方法
  static sing() {
    console.log(`singing now !`)
  }
}
Person.sing()  // singing now !
console.log(Person.age);  // 20

类实现接口 implements​

interface LiveProp {
  name: string,
  id: number,
  alert(): void;
}

class Live implements LiveProp {
  name: string
  id: number
  constructor(name: string, id: number) {
    this.name = name;
    this.id = id;
  }
  alert() {}
}

抽象类​

  • 抽象类一般作为基类来使用，只能被继承，不会用来直接实例化
  • 抽象类可以写入抽象方法，抽象方法不能具体实现
  • 子类继承抽象类后，抽象类的抽象方法必须在子类里面实现，否则报错
abstract class Animal {
  name: string
  constructor(name) {
    this.name = name
  }
  abstract eat()  // 抽象方法，在抽象类中不需要实现
}

class Dog extends Animal {
  constructor(name) {
    super(name)
  }
  eat() {
    console.log(this.name)
  }
}
const dog = new Dog('xiaohuang')
dog.eat()  // xiaohuang
